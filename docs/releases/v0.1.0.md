# MAIA v0.1.0 / MAIA v0.1.0

## Release summary / 发布摘要

MAIA v0.1.0 is the first public MVP release of the Multi-Agent Multi-Account Isolation Architecture for OpenClaw.  
MAIA v0.1.0 是面向 OpenClaw 的 Multi-Agent Multi-Account Isolation Architecture 的首个公开 MVP 版本。

It focuses on **per-user profile and memory isolation inside one workspace** using a **plugin + bundled skill** model.  
这个版本聚焦于通过 **plugin + 内置 skill** 模型，在一个 workspace 内实现 **按用户隔离的档案与记忆管理**。

## Highlights / 亮点

- **Regular plugin + hook MVP**  
  MAIA currently runs as a regular OpenClaw plugin with hooks rather than a custom context engine.  
  MAIA 当前以普通 OpenClaw 插件 + hooks 的形式运行，而不是自定义 context engine。

- **Per-user filesystem isolation**  
  Personal memory is organized under `people/user-id/` style folders instead of being mixed at the agent root.  
  个人记忆被组织在 `people/user-id/` 这类目录下，而不是混在 agent 根目录。

- **Bundled companion skill**  
  The MAIA skill is now bundled inside the plugin so fresh installs are closer to a single package.  
  MAIA skill 现在直接内置在插件中，让全新安装更接近单包分发。

- **Fresh install guide included**  
  The repository now includes a documented clone → link-install workflow for new OpenClaw environments.  
  仓库现在包含适用于新 OpenClaw 环境的 clone → link-install 安装指南。

- **Real compatibility validation**  
  MAIA has been validated against OpenClaw `2026.3.13`.  
  MAIA 已针对 OpenClaw `2026.3.13` 完成兼容性验证。

## Included in this release / 本次发布包含

- plugin manifest and runtime scaffold  
  插件 manifest 与运行时骨架
- `before_prompt_build`-based MAIA hook behavior  
  基于 `before_prompt_build` 的 MAIA hook 行为
- per-user path resolution and bootstrap scaffold  
  按用户路径解析与建档骨架
- bundled `openclaw-maia` skill  
  内置的 `openclaw-maia` skill
- install, fresh-install, architecture, release, and example docs  
  安装、fresh-install、架构、发布、示例等文档

## Known limitations / 已知限制

- OpenClaw does not currently install this plugin directly from a GitHub repo URL.  
  OpenClaw 当前还不能直接从 GitHub 仓库 URL 安装这个插件。
- The recommended install flow is still: clone repo → link-install plugin.  
  当前推荐安装方式仍然是：clone 仓库 → link-install 插件。
- MAIA is not yet a custom context engine.  
  MAIA 目前还不是自定义 context engine。
- Metadata capture and bootstrap richness are still minimal.  
  元数据采集与建档内容目前仍偏最小化。
